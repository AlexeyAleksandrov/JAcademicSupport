Этот проект демонстрирует работу теории свидетельств Демпстера-Шафера (DST) для агрегации данных из разнородных источников при обновлении образовательных программ.

## 10 демонстрационных примеров

### 📋 Как читать исходные данные

В каждом примере есть таблица с данными от трёх источников:

| Источник | Всего | По кластеру | Средняя оценка | Согласованность |
|----------|-------|-------------|----------------|-----------------|
| Вакансии | 1000 | 800 | 0.90 | 1.0 |
| Эксперты | 20 | 15 | 0.95 | 0.9 |
| Прогнозы | 5 | 4 | 0.85 | 1.0 |

**Что означает каждая колонка:**

- **Всего** — общее количество записей этого типа в БД (все вакансии/эксперты/прогнозы)
- **По кластеру** — сколько из "Всего" относятся к конкретному кластеру (например, Python)
- **Средняя оценка** — насколько источник считает кластер важным (0-1)
- **Согласованность** — насколько согласны между собой источники одного типа (0-1)

**Почему где-то умножаем на согласованность, а где-то нет?**

Формула эффективного объёма: `n_eff = (По кластеру) × (Согласованность)`

- **Вакансии:** 800 × 1.0 = 800 — умножаем на 1, число не меняется
- **Эксперты:** 15 × 0.9 = 13.5 — умножаем на 0.9, получаем меньше
- **Прогнозы:** 4 × 1.0 = 4 — умножаем на 1, число не меняется

📌 **Согласованность всегда участвует!** Просто когда она = 1.0, результат не меняется.

**Подробнее про метрики:**

**Средняя оценка (0-1):** "Насколько сильно источник поддерживает кластер?"
- 0.9+ = "Очень важно, must have!"
- 0.5 = "Не уверен, 50/50"
- 0.1- = "Не важно, можно пропустить"

**Согласованность (0-1):** "Насколько источники одного типа согласны между собой?"
- 1.0 = "Все как один говорят одно и то же"
- 0.5 = "Мнения разделились, есть споры"
- 0.0 = "Полный раскол, половина за, половина против"

**Пример:** 15 экспертов, средняя оценка 0.95
- При согласованности 0.9: эффективный объём = 13.5, κ = 0.73 (надёжно)
- При согласованности 0.5: эффективный объём = 7.5, κ = 0.60 (сомнительно)
- При согласованности 0.1: эффективный объём = 1.5, κ = 0.23 (нельзя доверять!)

📌 Даже при высокой средней оценке, низкая согласованность делает данные ненадёжными!

**Как считается согласованность (для любознательных):**

Формула: `Согласованность = 1 − (Дисперсия / Макс. дисперсия)`

**Что такое дисперсия?** Разброс мнений экспертов:
```
Пример 1 (все согласны):  0.94, 0.95, 0.96 → дисперсия = 0.0001
Пример 2 (разброс):       0.99, 0.50, 0.01 → дисперсия = 0.16
```

**Максимальная дисперсия = 0.25** — это когда половина экспертов говорит 1.0, половина 0.0

**Считаем:**
```
Все согласны:  1 − (0.0001/0.25) = 0.999 ≈ 1.0
Средний разброс: 1 − (0.01/0.25) = 0.96
Сильный разброс: 1 − (0.16/0.25) = 0.36
Полный раскол: 1 − (0.25/0.25) = 0
```

**Зачем нам дисперсия? Ключевой пример:**

Представьте, 15 экспертов дали **среднюю оценку 0.95**:

**Вариант А (все согласны):**
```
Оценки: 0.94, 0.95, 0.95, 0.96... 
→ Дисперсия = 0.0001 (маленькая)
→ Согласованность = 1.0
→ Вывод: "Python критически важен!"
```

**Вариант Б (средняя тоже 0.95!):**
```
7 экспертов: 1.0, 1.0, 1.0, 0.99, 0.98, 0.97 (за Python)
8 экспертов: 0.10, 0.15, 0.20, 0.12, 0.08, 0.18 (против Python)
→ Дисперсия = 0.20 (огромная!)
→ Согласованность = 0.20
→ Вывод: "Эксперты спорят, доверять нельзя!"
```

📌 **Без дисперсии мы бы приняли одинаковое решение по средней оценке 0.95, но в Варианте Б данные ненадёжны!**

**Как быстро оценить дисперсию:**
```
max(оценки) − min(оценки):
< 0.05 → отлично (дисперсия ~0.001)
< 0.15 → хорошо (дисперсия ~0.01)
< 0.30 → сомнительно (дисперсия ~0.04)
> 0.50 → катастрофа (дисперсия >0.20)
```

### 📐 Шаг 2: BPA — что такое и зачем нужно?

**BPA (Basic Probability Assignment)** — распределение уверенности между тремя вариантами:
- **m(T)** — "Кластер важен (включить)"
- **m(U)** — "Не знаю / не уверен"  
- **m(F)** — "Кластер не важен (исключить)"

**Зачем это нужно:**

Мы посчитали κ = 0.982 (коэффициент уверенности). Но **во что именно** мы уверены? Что Python важен на 98.2%? Это было бы слишком категорично.

**Формула BPA:**
```
m(T) = κ × (средняя оценка) × (доля релевантных данных)
m(U) = 1 - m(T)
m(F) = 0 (для внешних источников — они не утверждают "не важно")
```

**Пример для вакансий:**
```
m(T) = 0.982 × 0.90 × (800/1000) = 0.707
m(U) = 1 - 0.707 = 0.293
```

**Что это значит:**
- "Я на 70.7% уверен, что Python важен"
- "29.3% — неопределённость (я не знаю точно)"
- "Я не утверждаю, что Python бесполезен"

**Что даёт BPA:**
1. **Честно выражаем неуверенность** — не "Python важен на 98%", а "70% уверенности + 30% сомнений"
2. **Готовит к комбинированию** — на следующем шаге мы "сложим" мнения от разных источников
3. **Показывает вклад каждого источника:**
   - Вакансии: m(T) = 0.707 (высокий вклад, данных много)
   - Эксперты: m(T) = 0.520 (средний вклад, данных меньше)
   - Прогнозы: m(T) = 0.453 (ниже вклад, данных мало)

### 🎯 Шаг 3: Дисконтирование — учитываем качество источников

**Проблема:** Источники разные по надёжности. Вакансии содержат дубли, прогнозы часто ошибаются, а эксперты — профессионалы.

**Решение:** Умножаем BPA на вес надёжности w:
- **Эксперты:** w = 0.9 (высокая надёжность)
- **Вакансии:** w = 0.8 (хорошо, но есть дубли)
- **Прогнозы:** w = 0.6 (часто ошибаются)

**Формула:**
```
m'(T) = w × m(T)          — уверенность падает
m'(U) = 1 - w×(1 - m(U))  — неопределённость растёт
```

**Пример:**
```
Вакансии до:  m(T) = 0.707, m(U) = 0.293
После (w=0.8): m'(T) = 0.566, m'(U) = 0.434

Потеря = 14.1% ушло в неопределённость
```

**Зачем это нужно:**
1. ✅ "Штрафуем" ненадёжные источники (прогнозы сильно проседают)
2. ✅ Эксперты получают больший относительный вес
3. ✅ Готовим к комбинированию — теперь все источники "в одной весовой категории"

### 🔗 Шаг 4: Комбинирование — слияние мнений

**Проблема:** У нас три источника, каждый со своим мнением. Как получить одно итоговое?

**Правильный подход:** Не усреднять, а использовать правило Демпстера-Шафера!

**Формула (упрощённо):**
```
m₁₂(T) = [m₁(T)×m₂(T) + m₁(T)×m₂(U) + m₁(U)×m₂(T)] / (1 - K)
```

**Где K (коэффициент конфликта):**
- K = 0 → источники согласны
- K > 0.4 → источники сильно противоречат!

**Пример на наших данных:**
```
Шаг 1: Вакансии ⊕ Эксперты
- До: вакансии 56.6%, эксперты 46.8%
- После: 76.9% ← уверенность выросла! Источники подтвердили друг друга

Шаг 2: (Результат) ⊕ Прогнозы
- До: результат 76.9%, прогнозы 27.2%
- После: 83.2% ← финальная уверенность
```

**Зачем это нужно:**
1. ✅ Усиливаем уверенность при согласии (56.6% + 46.8% → 76.9%)
2. ✅ Выявляем конфликты (K > 0.4 сигнализирует о проблеме)
3. ✅ Получаем единое итоговое мнение из множества источников

**Как считается K (коэффициент конфликта):**
```
K = m₁(T) × m₂(F) + m₁(F) × m₂(T)
```

K — это сумма "несовместимых" комбинаций, когда один источник говорит "ВАЖНО", а другой "НЕ важно".

**Пример K = 0 (нет конфликта):**
```
Вакансии:  m(T) = 0.566, m(F) = 0
Эксперты: m(T) = 0.468, m(F) = 0

K = 0.566 × 0 + 0 × 0.468 = 0
//     ↑       ↑  ↑      ↑
//  m₁(T)×m₂(F)  m₁(F)×m₂(T)
```
Никто не говорит "не важно" → конфликта нет!

**Пример с конфликтом (из Примера 2: Blockchain):**
```
Вакансии:  m(T) = 0.6, m(F) = 0      ← "Blockchain важен!"
Эксперты: m(T) = 0,  m(F) = 0.5    ← "Blockchain НЕ важен!"

K = 0.6 × 0.5 + 0 × 0 = 0.3 (30% конфликта!)
```

**Как интерпретировать K:**
| K | Значение |
|---|----------|
| 0 | ✅ Нет конфликта |
| < 0.2 | ⚠️ Незначительный |
| 0.2 - 0.4 | ⚠️ Умеренный |
| > 0.4 | 🔴 Высокий — требуется экспертиза! |

**Примечание:** В текущих данных m(F)=0, потому что внешние источники (вакансии, эксперты, прогнозы) не дают отрицательных оценок. Чтобы получить m(F) > 0 и конфликты, нужны источники, которые говорят "не важно": опросы выпускников, аналитика устаревших технологий, HR-данные об увольнениях. Подробнее в EXAMPLES_GUIDE.md.

### 📊 Шаг 5: BetP и показатель дефицита

**Проблема:** У нас есть m(T)=0.832 и m(U)=0.168 — две цифры. Как получить одно число для сравнения с текущей программой?

**BetP (вероятностная мера):**
```
BetP = m(T) + m(U)/N = 0.832 + 0.168/25 = 0.839
```
📌 "Размазываем" неопределённость между 25 кластерами. Каждый получает по 0.007.

**Показатель дефицита:**
```
Δ = BetP - supply = 0.839 - 0.25 = 0.589
```
- BetP = 83.9% — внешний мир требует
- supply = 25% — в программе есть
- Δ = 58.9% — дефицит огромный!

**Интерпретация Δ:**
| Δ | Значение | Решение |
|---|----------|---------|
| > 0.15 | Дефицит | ✅ Усилить |
| -0.15 до 0.15 | Баланс | ✅ Сохранить |
| < -0.15 | Избыток | ⚠️ Ослабить |

**Зачем нужно:**
1. Преобразуем неопределённость в одно число для сравнения
2. Сравниваем внешние требования с текущей программой
3. Автоматически принимаем решение (усилить/сохранить/ослабить)

## 🔬 Как считается λ (лямбда) автоматически?

**Проблема:** Мы взяли λ = 15 для вакансий, 5 для экспертов, 2 для прогнозов. Но откуда эти числа?

**Ответ:** Они вычисляются автоматически из ваших данных!

**Метод 1: По медиане (самый простой)**
```
Берём все кластеры, собираем объёмы данных:
[800, 450, 320, 150, 15, 8, 5, 3, ...] — вакансии

Медиана (середина выборки) = 95
λ = медиана / 10 = 9.5 ≈ 10 (или округляем до 15)

📌 **Почему делим на 10?** Чтобы при типичном объёме κ ≈ 0.9:
```
κ = 95 / (95 + 9.5) = 95 / 104.5 ≈ 0.91 ✅ высокая уверенность!

Если бы делили на 5: κ = 95 / (95 + 19) = 0.83 (ниже доверие)
Если бы делили на 20: κ = 95 / (95 + 4.75) = 0.95 (слишком строго)
```

Число 10 подобрано эмпирически для баланса между доверием и осторожностью.

**Метод 2: По калибровке (самый точный)**
```
Пробуем разные λ на исторических данных:
λ = 5 → точность предсказаний = 72%
λ = 10 → точность = 85%
λ = 15 → точность = 91% ✅ ← выбираем!
λ = 20 → точность = 88%
```

**Почему разные λ для разных источников?**
| Источник | Типичный объём | λ | Почему так |
|----------|----------------|---|------------|
| Вакансии | 100-1000 | 15 | Вакансий много, нужен большой объём для уверенности |
| Эксперты | 5-20 | 5 | Экспертов мало, но мнение ценное |
| Прогнозы | 2-10 | 2 | Прогнозов редко много, 2 уже достаточно |

📌 **Важно:** λ не константа навсегда! Пересчитывается раз в год при обновлении данных.

## Ключевые метрики

-   **m(T)** — уверенность во включении (0-1)
-   **m(U)** — неопределённость (0-1)
-   **m(F)** — уверенность в исключении (0-1)
-   **κ (kappa)** — коэффициент уверенности от объёма данных (0-1)
-   **w** — вес надёжности источника (константы из статьи)
-   **K** — конфликт между источниками (0-1)
-   **Bel** — функция доверия
-   **Pl** — функция правдоподобия
-   **BetP** — вероятностная мера для ранжирования
-   **Δi** — показатель дефицита (BetP − supply)
-   **τΔ, τK, τΘ** — пороги принятия решений

## Формулы

### Коэффициент уверенности κ

```
κ = n_eff / (n_eff + λ)
```

где n_eff — эффективный объём данных, λ ≈ 10-20

### Дисконтирование

```
m'(T) = w × m(T)
m'(U) = 1 − w × (1 − m(U))
```

### Комбинирование по Демпстеру

```
K = m₁(T) × m₂(F) + m₁(F) × m₂(T)
m₁₂(T) = (m₁(T) × m₂(T) + m₁(T) × m₂(U) + m₁(U) × m₂(T)) / (1 − K)
m₁₂(U) = (m₁(U) × m₂(U)) / (1 − K)
```

### Показатель дефицита

```
Δi = BetP(Ci) − supplyi
```
